# Observability Initiative — Project Summary

## Project Overview

The Observability Initiative project was initiated in July 2024 and is led by Derek Washington. The core team includes John Doe, David Kim, Ryan O'Brien.

Goal: Implement distributed tracing and unified monitoring across all microservices.

## Technical Architecture

The technical stack includes Jaeger, OpenTelemetry, Datadog, Prometheus.

Jaeger was selected for distributed tracing with service dependency visualization.

OpenTelemetry was selected for vendor-neutral instrumentation standard.

Datadog was selected for unified dashboards and alerting across infrastructure.

Prometheus was selected for metric collection for Kubernetes-native workloads.

## Key Decisions

1. Adopted trunk-based development (September 2024)
1. Approved Istio for service mesh (October 2024)

- Ongoing: Active development and iteration
